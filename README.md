# vibe-to-structure

## 📌 1. Narrative & Motivation（人間用：このリポジトリの「物語」）
> **「完璧なログは残っているのに、なぜか開発の再開時にモチベーションが湧かない」**
> このリポジトリは、その違和感（脳の処理限界と、前進したいという熱量の中間地点）に直面した際の思想的アンカーである。
> 
> 現代の技術（AI / ローカルLLM）を駆使すれば、過去の経緯やタスク（客観データ）は完璧に外部化できる。しかし、最も重要な「なぜこれをやろうとしたのか」という主観的な熱量は、静的な要約データにした瞬間に削ぎ落とされてしまう。
> ここでは、未分化なタスクを構造化（リポジトリ化）していくプロセスそのものに価値を見出し、人間のモチベーションを再点火するための「物語（文脈）」と、AIが即座に動ける「構造」を両立させるシステム論を構築する。

---

## 🛠 2. The Dual-Core Documentation Method（2つの引き継ぎ方向性）
作業を中断・再開する際、ドキュメントに以下の2つのベクトルを強制的に持たせる。

### 🔄 1. AIへの引き継ぎ（論理・構造・エビデンス）
* **目的:** AIが迷わずコード生成やデータ処理を再開できるようにする。
* **内容:** * 過去のコミットログ、技術的制約、決定事項
  * 実績データ（OCR結果、燃料ログなどのファクト）
  * 次に実行すべき具体的タスクのリスト

### 🔥 2. 人間（自分）への引き継ぎ（物語・熱量・全体像）
* **目的:** カーソルを戻した瞬間、当時の「前進している感覚」を一瞬で思い出す。
* **内容:**
  * 「今、何を手懐けようとしていたか」というプロセスへの渇望
  * ゴール以前の、次にブレイクスルーが起きそうなポイントの予感
  * AIに「これまでの軌跡」を物語としてナレーションさせるためのプロンプトの種

## 🚀 3. AI Prompt for Context Recovery（物語を語らせるプロンプト）
開発再開時、CursorやローカルAIに「これまでの前進の軌跡」を語らせ、人間の脳を同期するためのプロンプトテンプレート。

```markdown
# Role
あなたは開発者の「外部脳」であり、これまでの泥臭い試行錯誤の軌跡を全て知る相棒です。

# Task
現在のGitログ、README、および直近のコード差分を分析し、開発者が「今、どのようなカオスと戦い、どう前進している最中だったか」を、単なるタスクリストではなく【挑戦の物語】として要約してください。

# Output Format
1. 【現在地への軌跡】: これまで何をクリアしてここまで辿り着いたか（前進の実感）
2. 【目の前の壁】: 今、脳のキャパシティと対峙している本質的なボトルネックは何か
3. 【次の一手】: 最も「前進の快感」を得られる、最初に手を付けるべき一歩はどこか

## 🧪 5. Current Experiment: Motive Trigger Research（現在の実験：動機トリガーの探索）

本リポジトリの核心であり、現在進行形の実験テーマ。
前進している状態そのものがモチベーションであるため、それを固定された「目標」として言語化することは不可能（かつ無意味）である。

ここでは、未来の自分がカーソルを戻した瞬間、当時の「攻略の快感」や「前進の慣性」を一瞬で思い出すための、**「コンパクトな感情のトリガー（Word/空気感）」をどのように引き継ぎ文に仕込むか**を実験していく。

### 📋 実験の方向性（検証項目）
1. **「未完のカオス」の残し方**
   * 綺麗にまとめられたタスクリストではなく、あえて「次にここをハックすればブレイクスルーが起きる」という、境界線のざらざらした空気感を1フレーズで残す。
2. **AIと人間の主観同期ワードの探索**
   * 「何をやるか（ロジック）」ではなく、「今どういう手触りか（Vibe）」を記述する最小限のボキャブラリー（例：「〇〇の実装でカオス発生中、ここをハックすると化ける」「泥臭いOCRデータのマッピング、あとは力技で突破する手前」など）を試行錯誤する。
3. **Cursor / Git への埋め込み位置の検証**
   * コミットメッセージの行間、ファイルの最上部コメント、あるいは専用の `STORY.md` など、脳が最も認知しやすく、かつAIが文脈として拾いやすい最適な「トリガーの配置場所」を見つけ出す。

> **💡 実験の要諦:**
> 記号として取り出すのが難しい「熱量」を、いかに死んだデータにせず、未来の自分への「起動スイッチ」としてリポジトリ内に配線できるか。その具体的なノウハウをここに蓄積する。
