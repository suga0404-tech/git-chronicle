# git-chronicle

## 📌 1. 思想：脳の外部化における熱量の引き継ぎ
現在の技術は、過去のログやタスクといった「論理（客観データ）」を完璧にアーカイブできるが、「なぜそれをやろうとしたのか」という人間の「熱量（主観）」は、データ化した瞬間に揮発してしまう。

本プロジェクトは、定まっていないタスクをリポジトリとして扱い、徐々に構造へ落とし込むプロセスの中で、「AIができること」と「人間がやらねばならないこと」を明確に分離し、未来の自分へ文脈を引き継ぐための実験場である。

---

## 🛠 2. AIによるアプローチ：バージョンの物語化（プロンプト）
蓄積された客観データ（Gitログや差分）をベースに、AIにこれまでの試行錯誤を「物語」としてナレーションさせ、人間の脳を同期させる試み。これはAIが十分に機能できる領域である。

開発再開時、CursorやローカルAIに以下のプロンプトを入力し、コンテキストを回復する。

```text
# Role
あなたは開発者の「外部脳」であり、これまでの泥臭い試行錯誤の軌跡を全て知る相棒です。

# Task
現在のGitログ、README、および直近のコード差分を分析し、開発者が「今、どのようなカオスと戦い、どう前進している最中だったか」を、単なるタスクリストではなく【挑戦の物語】として要約してください。

# Output Format
1. 【現在地への軌跡】: これまで何をクリアしてここまで辿り着いたか（前進の実感）
2. 【目の前の壁】: 今、脳のキャパシティと対峙している本質的なボトルネックは何か
3. 【次の一手】: 最も「前進の快感」を得られる、最初に手を付けるべき一歩はどこか
